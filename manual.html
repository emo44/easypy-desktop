1. Introduction

EasyPy Desktop is a professional Rapid Application Development (RAD) environment. It allows you to build native Windows applications using the power of Python 3 and the PySide6 (Qt) framework, but with a visual drag-and-drop interface.

Key Philosophy: "Visual for Design, Python for Logic". Unlike blocks-based tools, here you write real Python code, giving you unlimited power.

Installation & Setup

Layouts & Anchors (Responsive Design)

Creating apps that look good on different screen sizes is handled via the Anchor property in the Right Panel.

‚ö†Ô∏è
Tip: Anchors only work when the window is resized. In the editor, you see the fixed size.
  • Top-Left (Default): Stays fixed to the top-left corner.
  • Top-Right: Moves with the right edge. Good for "Close" buttons.
  • Stretch-Width: The widget grows horizontally as the window widens. Good for Title Bars or Text Inputs.
  • Stretch-Both: The widget grows in all directions. Ideal for background images, maps, or main text areas.

4. Python Scripting Guide

When you write code in a button, it executes within the application's scope. You don't need `def main():`. Just write the logic directly.

Internal API

These functions are available globally in your scripts:

FunctionDescription
find_widget("id")Returns the widget object. Use .text(), .setText(), .show(), etc.
goto_page("Name")Switches the view to another page defined in the "Create" tab.
show_message("Title","Msg")Shows a native popup alert.
close_application()Exits the program.
selfAccess to the main `QMainWindow`.

State Management (ctx)

Variables declared in a script die when the script finishes. To store data persistently (like a score, username, or toggle state), use the dictionary ctx.

Example: A Counter

# Check if variable exists, if not, initialize it
if 'count' not in ctx:
    ctx['count'] = 0

# Increment
ctx['count'] += 1

# Update UI
find_widget("lbl_counter").setText(str(ctx['count']))

Debugging Console

If your script fails, nothing might happen. To see errors:

  1. Run the project with F5.
  2. A bottom panel "Console / Debugger" will appear.
  3. print("hello") output and Python Errors (Tracebacks) will appear there.

5. Functions & Subroutines

In EasyPy Desktop, you don't use "Modules" or "Forms" in the Visual Basic style. Instead, you use Python's dynamic nature to organize your code. This section explains how to define reusable logic.

Local Functions

If you have complex logic inside a single button, you can define helper functions right inside the event script. These functions exist only while that event is running.

Example: Calculating Taxes

# Define a local helper
def calculate_total(price):
    return price * 1.21

# Main Logic
input_val = find_widget("input_price").text()
if input_val:
    val = float(input_val)
    final_price = calculate_total(val)
    find_widget("lbl_result").setText(f"${final_price:.2f}")

Global Subroutines (Public Subs)

To create a function that can be called from any page or widget (like a Public Sub in a Module), you should store the function in the ctx dictionary.

Step A: Define it (e.g., in "Startup" or "Master Page")

Write this code in a button or script that runs once at the beginning:

# Define the function
def log_system_event(message):
    import datetime
    time = datetime.datetime.now().strftime("%H:%M:%S")
    print(f"[{time}] {message}")

# Store it in ctx to make it global
ctx['log'] = log_system_event

Step B: Call it (from any other widget)

# Now you can use it anywhere
if 'log' in ctx:
    ctx['log']("User clicked Save")

Attaching to 'self' (Advanced)

For cleaner syntax, advanced users can attach functions directly to the main window instance (self). This creates methods that persist as long as the app is running.

def open_google_func():
    import webbrowser
    webbrowser.open("https://google.com")

# Attach to application window
self.open_google = open_google_func

# Usage later:
# self.open_google()

6. Advanced Topics

SQLite Databases

Use the SQLite Manager Plugin (found in the "Plugins" tab). It provides a high-level Grid widget.

Workflow:

  1. Drag SQLite Grid to canvas. ID: my_grid.
  2. Create a button "Connect". Use the wizard "Initialize DB".
  3. Create inputs for data (e.g., Name, Phone).
  4. Create a button "Add". Use the wizard "Insert Record" and map the inputs to columns.

HTTP Requests

The library requests is pre-installed. You can call APIs easily.

import requests

try:
    r = requests.get("https://api.coindesk.com/v1/bpi/currentprice.json")
    data = r.json()
    price = data['bpi']['USD']['rate']
    
    find_widget("lbl_price").setText(f"Bitcoin: ${price}")
except Exception as e:
    show_message("Error", str(e))

File System & Paths

When compiling to EXE, files are unpacked to a temp folder. To load assets correctly in code, use resource_path() if using the internal API helpers, OR build paths dynamically.

Best Practice: Keep everything in the assets folder.

7. Plugin SDK

You can extend EasyPy by writing Python scripts in the plugins_user folder.

Minimal Plugin Structure (my_plugin.py):

from PySide6.QtWidgets import QPushButton

# 1. Metadata
PLUGIN_METADATA = {
    'name': {'en': "My Button"},
    'version': "1.0",
    'author': "Me"
}

# 2. The Widget Class
class MyButton(QPushButton):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.setText("Hello Plugin")

# 3. Initialization Hook
def initialize_plugin(mw):
    # id, class, width, height, default_signal
    mw.register_plugin_tool('my_btn', MyButton, 100, 30, 'clicked')

8. Building .EXE

Press F9 to compile.

The "Smart Compiler"

EasyPy automatically scans your code to detect dependencies.

  • If you use requests, it includes SSL libraries.
  • If you use sqlite3, it includes database drivers.
  • If you use WebView or Media, it includes the heavy Qt WebEngine/Multimedia DLLs.
‚ö†Ô∏è
Antivirus Warning: Compiled Python EXEs (PyInstaller) are often flagged as false positives by Windows Defender. You may need to exclude the folder or sign the EXE.

Troubleshooting

"Runtime Python not found"

The compiler needs the runtime folder next to the main executable. If you moved the exe, move the folder too.

"ImportError: No module named..." in EXE

If you import a non-standard library (e.g., pandas), you must install it into the runtime python environment using the "Manage Libraries" tool in the "Project" menu.

Video/Audio not playing

Ensure the multimedia backend codecs are installed on Windows (K-Lite Codec Pack often helps if system codecs are missing). EasyPy tries to bundle the Qt Multimedia DLLs, but it relies on the OS for decoding.
